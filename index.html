Git Commands

Регистрация в GIT

git --versionПроверка версии GIT
git configПросматривает и устанавливет параметры
git config --global user.name "Имя"Задать имя пользователя
git config --global user.email Ваша @ почтаЗадать почту пользователя
git config --listПолучение списка всех настроек

Работа в GIT

git initСоздание нового репозитория
git statusОтображение текущего статуса файлов
git status --untracked-files=allОтображение текущего статуса файлов, включая файлы в папках
git add Имя файла Подготовка файлов перед коммитом. Добавляется указанный файл
git add .Подготовка файлов перед коммитом. Добавляются все файлы проекта. Эту команду можно повторять вместо команды "git add Имя файла" после создания нового файла, если предыдущие файлы уже добавлены в коммит
git rm --cached Имя файлаОтменяет подготовку файла перед коммитом, оставив его при этом в рабочем каталоге
git config --global core.autocrlf falseРешает ошибку "warning: CRLF will be replaced by LF."
git commit -m "комментарий"Создание коммита. Позволяет задать комментарий к коммиту
git commit -a -m "комментарий"Создание коммита. Индексирует каждый уже отслеживаемый на момент коммита файл, позволяя нам обойтись без git add
git commit --amend --no-editОбновляем коммит, текст коммита не меняется
git commit --amend -m "комментарий"Обновляем коммит с изменением текста
git add . && git commit -m "комментарий"Одновременно делает команды "git commit -m" и "git add .". Полезна когда одновременно создаем новый файл в проетке и вносим изменения в уже существующий и добавленный в коммит файл.
git reset --soft HEAD^Команда отменит последний коммит (но не изменения, они сохранятся). Все это работает, если вы не опубликовали свои изменения командой "git push"
git reset --hard HEAD~1 Команда полностью удаляет последний коммит (можно удалять любой комит, поменяв цифру 1 на порядковый номер нужного коммита). Все это работает, если вы не опубликовали свои изменения командой "git push"
git restore Имя файлаВостановить конкретный файл из последнего коммита
git restore .Востановить все файлы из последнего коммита
git logПросмотр истории изменений. Если список очень большой, для выхода нажать Q
git log --pretty=format:"%h - %an, %ar - %s"Изменение формата вывода коммитов. %h – показывает короткий хеш-код коммита %an – автор коммита %ar – время, когда был сделан коммит %s – комментарий коммита
git log --since=2.weeksКоманда с данным параметром выводит все коммиты за последние 2 недели
$ git log -p -n–p – показывает diff (дельту, разницу), внесенную каждым коммитом. –n – ограничит вывод "n"-последними записями.
git checkout короткий хэш коммитаПереход на определенную версию проекта по короткому хэшу коммита
git checkout masterВозврат к последнему актуальному коммиту ветки master
git branch новое имя веткиСоздать новую ветку с указанным именем, без переключения на нее
git checkout имя ветки Переход на ветку по имени
git checkout -b новое имя ветки Создать ветку и перейти на нее
git branch Команда git branch без параметров отобразит список веток и укажет активную в данный момент ветку
git branch -vКоманда с ключем –v отобразит список веток c коротким хэш-кодом последнего коммита и комментарием к нему
git merge имя веткиСделать слияние веток. Нужно перейти на ветку, куда будем делать слияние, а в команде прописать имя ветки откуда будем делать слияние. При кофликте слияний, лучше всего править в VSCode (встроенная утилита Merge Editor)
git merge --abortОтменить слияние веток
git remoteОтображение подключенных к локальному проекту удаленных серверов
git remote -vПросмотр соответствия псевдонима и адреса репозитория GIT. fetch – путь, откуда мы будем забирать файлы. push – путь, куда мы будем загружать файлы.
git remote rm origin Удаление ссылки на удаленный сервер с именем origin, связанный с локальным репозиторием. Только после этого можно подключить новый репозиторий GitHub
git push -u имя сервера имя веткиПервая выгрузка файлов активной ветки на удаленный репозиторий (первичная отпрака, нужно указать имя активной ветки)
git pushПоследующие отправки активной ветки на удаленный сервер
git clone имя сервераКлонирование репозитория с удаленного сервера
git pullЗагрузка удаленных изменения в локальный репозиторий
git branch -aВывод информации о всех ветках репозитория, включая удаленные
git help lofОткрывает FAQ (работает оффлайн)
