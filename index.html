Git Commands

Регистрация в Гите

- git --versionПроверка версии GIT
- git configПросматривает и устанавливет параметры
- git config --global user.name "Имя"Задать ваше имя
- git config --global user.email Ваша @ почтаЗадать вашу почту
- git config --listПолучение списка всех настроек

Работа в Гите

- git initСоздание нового репозитория
- git statusОтображение текущего статуса файлов
- git status --untracked-files=allОтображение текущего статуса файлов, включая файлы в папках
- git add Имя файла Подготовка файлов перед коммитом. Добавляется указанный файл
- git add .Подготовка файлов перед коммитом. Добавляются все файлы проекта
- git rm --cached Имя файлаОтменяет подготовку файла перед коммитом, оставив его при этом в рабочем каталоге
- git config --global core.autocrlf falseРешает ошибку "warning: CRLF will be replaced by LF."
- git commitСоздание коммита
- git commit -m "комментарий"Создание коммита. Позволяет задать комментарий к коммиту
- git commit -a -m "комментарий"Создание коммита. Индексирует каждый уже отслеживаемый на момент коммита файл, позволяя нам обойтись без git add
- git commit --amend --no-editОбновляем коммит, текст коммита не меняется
- git commit --amend -m "Текст"Обновляем коммит с изменением текста
- git add . && git commit -m "updated 3 html"
- git reset --soft HEAD^Команда отменит последний коммит (но не изменения, они сохранятся). Все это работает, если вы не опубликовали свои изменения
- git reset --hard HEAD^ Последний коммит можно полностью удалить. Все это работает, если вы не опубликовали свои изменения
- git revert commit-sha1Отменяет опубликованый коммит. Потом опубликовать его командой git push
- git restore filename
- git restore .
- git logПросмотр истории изменений
- git log --pretty
- git checkout hash-commit переход на коммит
- git checkout branch-name переход на ветку по имени
- git checkout -b new-branch-name создаем ветку и переходим на нее
- git branch
- git branch -v
- git merge branch-name
- git remote
- git remote -v
- git remote rm origin
- git push -u origin master
- git pushОтправка на удаленнній сервер
- git cloneКопирование с удаленного сервера
- git pullСкачивание с удаленного сервера
